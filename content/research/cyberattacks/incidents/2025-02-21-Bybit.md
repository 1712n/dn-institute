## Summary

On February 21, 2025, [Bybit](https://www.bybit.com/), one of the world's largest cryptocurrency exchanges, suffered the [largest cryptocurrency theft in history](https://www.trmlabs.com/resources/blog/the-bybit-hack-following-north-koreas-largest-exploit), losing approximately 401,347 ETH (~$1.5 billion) from its Ethereum cold wallet. The attack was carried out by the [Lazarus Group](https://www.fbi.gov/), a North Korean state-sponsored hacking organization, as [confirmed by the FBI](https://www.ic3.gov/PSA/2025/PSA250226) on February 26, 2025. The attackers compromised a developer machine at [Safe{Wallet}](https://safe.global/), the multisig wallet provider used by Bybit, and injected malicious JavaScript into the Safe{Wallet} web interface. This tampered code altered transaction data shown to Bybit's multisig signers, causing them to unknowingly approve a transaction that replaced the cold wallet's smart contract implementation with one controlled by the attackers. The stolen funds were rapidly dispersed across [multiple wallets, decentralized exchanges, and mixing protocols](https://www.nccgroup.com/research-blog/in-depth-technical-analysis-of-the-bybit-hack/). Bybit CEO Ben Zhou [confirmed the exchange remained solvent](https://therecord.media/hackers-drained-bybit-crypto-exchange-hack) with all client assets backed 1:1 and launched a [recovery bounty program](https://www.bybit.com/en/press/post/bybit-launches-recovery-bounty-program-with-rewards-up-to-10-of-stolen-funds-bltcd3ebbb9445d5b74) offering up to 10% of recovered funds as a reward.

## Attackers

The attack was attributed to the **Lazarus Group** (also known as TraderTraitor and APT38), a [North Korean state-sponsored hacking group](https://www.picussecurity.com/resource/blog/fbi-north-korean-lazarus-group-bybit-crypto-heist) with a long history of targeting cryptocurrency platforms to fund North Korea's weapons programs. The FBI [issued a public service announcement](https://www.ic3.gov/PSA/2025/PSA250226) on February 26, 2025, formally attributing the hack to North Korea's TraderTraitor group. [Arkham Intelligence](https://x.com/arkham/status/1893033424224411885) also independently identified the attackers as the Lazarus Group shortly after the hack.

### Attack Method

The attack was a [multi-stage supply chain compromise](https://www.nccgroup.com/research-blog/in-depth-technical-analysis-of-the-bybit-hack/) targeting Bybit through its wallet infrastructure provider:

1. **Developer Machine Compromise:** The attackers first compromised the machine of a [Safe{Wallet} developer](https://x.com/safe/status/1894768522720350673), gaining the ability to inject malicious code into the Safe{Wallet} web application.

2. **Malicious JavaScript Injection:** A [JavaScript file was tampered](https://web.archive.org/web/20250219172905js_/https://app.safe.global/_next/static/chunks/pages/_app-52c9031bfa03da47.js) to include code that specifically targeted Bybit's cold wallet addresses. The code identified Bybit's authorized signers and wallet addresses, then silently replaced legitimate transaction data with a malicious `delegatecall` operation when a Bybit signer initiated a transaction.

3. **UI Spoofing:** When Bybit's signers reviewed and approved what appeared to be a routine cold-to-warm wallet transfer, the tampered JavaScript substituted the actual transaction with one that called a [malicious contract](https://etherscan.io/) (`0x96221423681A6d52E184D440a8eFCEbB105C7242`) via `delegatecall`. After signing, the code restored the original transaction data in the UI, hiding the manipulation from subsequent signers.

4. **Smart Contract Takeover:** The malicious transaction replaced the implementation contract of Bybit's multisig cold wallet, giving the attackers full control over the wallet's funds.

5. **Fund Extraction:** With control of the wallet, the attackers drained all 401,347 ETH and additional tokens (90,375 stETH, 15,000 cmETH, 8,000 mETH) from the cold wallet.

### Money Laundering

After the theft, the attackers executed a [coordinated laundering operation](https://www.trmlabs.com/resources/blog/the-bybit-hack-following-north-koreas-largest-exploit):
- Stolen stETH, cmETH, and mETH tokens were quickly swapped for ETH via DeFi protocols
- Funds were split across multiple wallets (initially 48+ addresses)
- Bitcoin mixers and peer-to-peer vendors were used to obfuscate the trail
- As of March 2025, Bybit CEO Ben Zhou reported [88.87% of stolen assets remained traceable](https://www.fintechweekly.com/magazine/articles/bybit-ceo-ben-zhou-says-most-funds-are-still-traceable)

## Losses

- **401,347 ETH** (~$1.4 billion at time of theft)
- **90,375 stETH** (Lido Staked ETH)
- **15,000 cmETH** (Mantle Staked ETH)
- **8,000 mETH** (Mantle ETH)
- **Total estimated loss: ~$1.5 billion**

Bybit [confirmed the exchange remained solvent](https://therecord.media/hackers-drained-bybit-crypto-exchange-hack) and that all client assets continued to be backed 1:1. The exchange secured bridge loans from partners to cover the shortfall while pursuing recovery efforts.

## Timeline

- **Days before February 21, 2025:** Lazarus Group compromises a Safe{Wallet} developer's machine and injects [malicious JavaScript](https://www.nccgroup.com/research-blog/in-depth-technical-analysis-of-the-bybit-hack/) into the Safe{Wallet} web application, specifically targeting Bybit's cold wallet addresses.
- **February 21, 2025, 14:13 UTC:** Attackers execute a [test transaction of 90 USDT](https://crystalintelligence.com/investigations/the-bybit-heist-how-the-hackers-took-control/) from the cold wallet to confirm bypass of the security system. Timing is deliberate — end of UAE business day with limited staff available.
- **February 21, 2025, 14:16 UTC:** The main theft occurs — [401,347 ETH, 90,375 stETH, 15,000 cmETH, and 8,000 mETH](https://crystalintelligence.com/investigations/the-bybit-heist-how-the-hackers-took-control/) are drained from Bybit's Ethereum cold wallet.
- **February 21, 2025, 14:29 UTC:** Attackers begin [dispersing stolen tokens](https://crystalintelligence.com/investigations/the-bybit-heist-how-the-hackers-took-control/) — a test ETH transaction to a new address, then stETH/cmETH/mETH tokens swapped for ETH via DeFi in batches of 10,000 across multiple addresses.
- **February 21, 2025, 15:51 UTC:** Bybit [publicly acknowledges the attack](https://x.com/Bybit_Official/status/1892965292931702929) on X (formerly Twitter), 82 minutes after the theft, assuring customers that other cold wallets are safe.
- **February 21, 2025:** Bybit CEO Ben Zhou holds a [livestream confirming 401,000 ETH stolen](https://therecord.media/hackers-drained-bybit-crypto-exchange-hack) and that the exchange remains solvent with all assets backed 1:1.
- **February 22, 2025:** Bybit launches its [recovery bounty program](https://www.bybit.com/en/press/post/bybit-launches-recovery-bounty-program-with-rewards-up-to-10-of-stolen-funds-bltcd3ebbb9445d5b74), offering up to 10% (~$140 million) of recovered funds as a reward and inviting blockchain analytics firms worldwide to assist.
- **February 26, 2025:** The FBI [issues a public service announcement](https://www.ic3.gov/PSA/2025/PSA250226) attributing the attack to North Korea's TraderTraitor (Lazarus Group).
- **March 2025:** Bybit CEO Ben Zhou reports that [88.87% of stolen funds remain traceable](https://www.fintechweekly.com/magazine/articles/bybit-ceo-ben-zhou-says-most-funds-are-still-traceable) despite laundering efforts through Bitcoin mixers and P2P vendors.

## Security Failure Causes

- **Supply Chain Compromise:** The attack exploited a vulnerability in Bybit's third-party wallet infrastructure provider ([Safe{Wallet}](https://safe.global/)) rather than Bybit's own systems directly. A [compromised developer machine](https://x.com/safe/status/1894768522720350673) at Safe{Wallet} allowed the attackers to inject malicious code into the production web application.
- **UI-Level Transaction Manipulation:** The multisig signing process relied on a [web interface that could be tampered](https://www.nccgroup.com/research-blog/in-depth-technical-analysis-of-the-bybit-hack/) with at the JavaScript level. Signers approved transactions based on what the UI displayed rather than independently verifying the raw transaction data on their hardware wallets or through alternative channels.
- **Insufficient Transaction Verification:** Bybit's signing process did not include [independent verification mechanisms](https://www.ocorian.com/knowledge-hub/insights/breaking-vault-lessons-custody-security-bybit-hack) (such as comparing transaction hashes across multiple independent interfaces or decoding raw calldata) that could have detected the discrepancy between displayed and actual transaction data.
- **Single Point of Trust in Wallet Provider:** Despite using a multisig setup requiring 3 signers, [all signers used the same Safe{Wallet} web interface](https://www.nccgroup.com/research-blog/in-depth-technical-analysis-of-the-bybit-hack/), creating a single point of failure. The malicious JavaScript targeted the shared application layer, bypassing the multi-signature security model entirely.
- **Ethereum Multisig Complexity:** Unlike blockchains with native multisig support, [Ethereum multisig wallets require smart contracts](https://crystalintelligence.com/investigations/the-bybit-heist-how-the-hackers-took-control/), adding complexity and attack surface. The use of `delegatecall` in the exploit allowed the attackers to change the wallet's implementation contract while preserving its address and state.
