# Market Health Reporter 🌰

Automated report generator for cryptocurrency market health metrics, featuring
Retrieval Augmented Generation (RAG) to enrich articles with real-time news
context.

## How it works

1. **Market data** is fetched from the [Cross-Market Surveillance API](https://cross-market-surveillance.p.rapidapi.com).
2. **RAG context** (optional) is retrieved from CryptoPanic and/or NewsAPI,
   embedded via [Cohere](https://cohere.com/), ranked by cosine similarity to a
   query derived from the market venue and trading pair, and the top-K snippets
   are injected into the LLM prompt.
3. **Report** is generated by GPT-4 using the market data, RAG context, and a
   structured prompt aligned with the [contribution guidelines](https://github.com/1712n/dn-institute/issues/277).

## CLI arguments

| Argument | Required | Description |
|---|---|---|
| `--llm-api-key` | ✅ | OpenAI API key |
| `--issue` | ✅ | GitHub issue number |
| `--comment-body` | ✅ | Trigger comment body (`pair:X, venue, start, end`) |
| `--github-token` | ✅ | GitHub token for posting results |
| `--rapid-api` | ✅ | RapidAPI key for market data |
| `--cohere-api-key` | ❌ | Cohere API key — enables RAG embedding & re-ranking |
| `--cryptopanic-api-key` | ❌ | CryptoPanic API key — enables news fetching for RAG |

## RAG pipeline 🌰

```
news APIs  →  raw articles
                  │
           Cohere embed
                  │
          cosine similarity vs query
                  │
           top-K snippets
                  │
          injected into prompt  →  GPT-4  →  article
```

## Example usage

```bash
python -m tools.market_health_reporter.market_health_reporter \
  --llm-api-key $OPENAI_API_KEY \
  --issue 123 \
  --comment-body "pair:btc-usdt, binance, 2024-01-01, 2024-01-07" \
  --github-token $GITHUB_TOKEN \
  --rapid-api $RAPID_API_KEY \
  --cohere-api-key $COHERE_API_KEY \
  --cryptopanic-api-key $CRYPTOPANIC_API_KEY
```
