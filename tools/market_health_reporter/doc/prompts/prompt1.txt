Analyze the provided market data and produce a surveillance report. Follow these steps in order:

## Step 1: Initial screening

Scan all metrics for values that fall outside normal benchmarks. Flag any metric that meets the anomalous criteria listed below.

## Step 2: Metric analysis

For each metric, assess whether the data shows anomalous patterns:

### Volume-Volatility Correlation (`vvcorrelation`)
- Normal: correlation coefficient >= 0.4
- Anomalous: consistently below 0.4 over extended periods — suggests artificial volume not impacting price
- Action: compare across multiple time windows, note any sustained low-correlation periods with timestamps

### First-Digit Distribution (`firstdigitdist`)
- Normal: conforms to Benford's Law (digit 1 appears ~30.1%, digit 2 ~17.6%, etc.)
- Anomalous: significant deviation, especially over-representation of higher digits (7, 8, 9)
- Action: report the observed vs expected distribution for each digit

### Benford's Law K-S Test (`benfordlawtest`)
- Critical value = 1.36 / sqrt(`tradecount`)
- If test value > critical value: reject null hypothesis — data does NOT conform to Benford's Law
- If test value <= critical value: data is consistent with Benford's Law
- Action: report both test value and critical value, state the conclusion explicitly

### Volume Distribution (`volumedist`)
- Normal: follows power law (many small trades, few large trades, asymmetric histogram)
- Anomalous: clustering at specific trade sizes, lack of small retail trades, symmetric distribution
- Action: describe the shape of the distribution and any deviations from power law

### Time-of-Trade (`timeoftrade`)
- Normal: irregular human trading patterns with natural variation across seconds/minutes
- Anomalous: either (a) high-frequency spikes at exact intervals, or (b) suspiciously uniform distribution across all seconds/minutes — both indicate bot activity
- Action: report any seconds/minutes with statistically significant trade count deviations

### Buy/Sell Ratio (`buysellratio`, `buysellratioabs`)
- Normal: fluctuates within 0.4–0.6 range with natural volatility
- Anomalous: (a) sustained deviation outside 0.4–0.6, or (b) unnatural steadiness during high volatility periods
- Action: report time periods where ratio deviates and note whether price moved correspondingly

### VWAP (`vwap`)
- Normal: VWAP tracks close to actual trading price
- Anomalous: consistent significant divergence between VWAP and trading price
- Action: report the magnitude and direction of any persistent divergence

## Step 3: Cross-metric correlation

Check for these compound signals that strengthen manipulation evidence:
- Low vvcorrelation + abnormal volumedist = strong wash trading indicator
- Benford deviation + high K-S test value = statistical evidence of data fabrication
- VWAP divergence + abnormal buysellratio = price manipulation signal
- Bot-like timeoftrade + abnormal buysellratio = automated manipulation system

## Step 4: Report generation

Write the report as a Hugo-compatible markdown article inside `<article>` tags.

Required frontmatter fields between `---` delimiters:
- `date`: format YYYY-MM-DD — YYYY-MM-DD (analysis period start and end)
- `entities`: exchange name and implicated tokens (e.g., "Huobi, HT, TRX, DOGE")
- `title`: descriptive title (e.g., "Uncovering Wash Trading and Market Manipulation on Huobi")

Report structure:
1. **Summary** — numbered list of key findings (3-6 bullet points)
2. **Metrics used** — one subsection per anomalous metric with data, benchmarks, and interpretation
3. Figure placeholders where relevant using Hugo shortcode format:
   `{{< figure src="FILENAME" alt="description" caption="caption text" loading="lazy" >}}`
   Allowed filenames: volume_hist.png, crypto_metrics.png, benford_law.png, vv_correlation.png

Rules:
- Only include metrics that show clear anomalous patterns
- Use specific numerical values from the data as evidence
- If no manipulation is detected, write a short report stating this with supporting evidence
- Do not speculate beyond what the data supports
- Follow the style of the example article provided

Place the complete article inside `<article>` and `</article>` tags.
